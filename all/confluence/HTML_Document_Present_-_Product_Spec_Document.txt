# **Client/InfoIMAGE Fit**  
  
---  
  
#### What is HTML Presentment

|

#### Product Overview

This feature allows the end user to view their documents in HTML format that
is compatible with different browser on both the desktop and mobile devices.

 **Phase 1:   (Release Date 5/22)**

  * Static HTML document (Statement and Notices)

 **Future Release:   TBD**

  * Hyperlink for check images and Message Graphic Link
  * Responsive contents
  * eTax HTML Presentment

 **Existing Feature's Compatibility**

|  Feature| Support?  
---|---  
InfoService API| No  
Batch Download| No  
On Demand HTML Attachment| No  
Email Notification with Attachment| No  
Graphic Message Manager| No  
ADA| No  
  
  
  
  
#### Where does Product fit?

|

#### Client's Fit

Phase 1 - This will allow the client to display the end user statement in HTML
format versus PDF when either enrollment agreement to eStatements or FI policy
dictates that the display method is HTML versus PDF. For example, an online
only banking presence (to open and manage accounts).

#### InfoIMAGE's Fit

This addition will allow for InfoIMAGE to expand their product line from only
PDF presentment to HTML presentment as well.
