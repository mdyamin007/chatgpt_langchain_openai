title: InfoConfig client property configuration guide
body:   1. If we are changing adding or updating a feature after saving the configuration we **MUST** check if that particular feature is saved as expected or not. It means we **MUST** explore all the sub properties of that **feature** as well.
  2. Most important thing is before change we **MUST** check the current payload and after changing we must check the expected payload for all the **APPLICATIONS**.
  3. We should mot update for multiple feature at once. We will update one feature at once.
